# Handoff Checklist for Paired Work

This checklist keeps our handoff quality consistent when one person continues work started by another colleague.

You should include current status, blockers, and the exact next validation step.

## Handoff sections

- What changed in this session
- Why the current approach was chosen
- Which risks still need confirmation

It is recommended to attach one runnable command instead of a broad test suite.

```bash
bun test tests/eval.test.ts
```

For asynchronous teams, you should tag the next reviewer and include a target response window.

See [handoff template](https://example.com/collab/handoff-template).
