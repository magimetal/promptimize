# Collaborative Review Protocol

Our reviewers and authors share responsibility for clear, shippable changes.

When a pull request is opened, you should post a short summary so teammates can align quickly.

## Review loop

- The reviewer should call out one high-risk area first.
- The author should reply with either a fix plan or evidence that behavior is already covered.
- Please keep comment threads focused on one decision at a time.

## Coordination rules

It is recommended that you use explicit ownership when multiple colleagues are involved.

- Handoff notes should include the next command to run.
- In order to avoid duplicate work, pair on unresolved design decisions for 15 minutes.

Use the [review etiquette guide](https://example.com/collab/review-etiquette) when tone becomes ambiguous.
